# SerpApi Plugin for Claude Code

A [Claude Code plugin](https://docs.anthropic.com/en/docs/claude-code/plugins) that gives Claude the ability to search Google, Amazon, Walmart, YouTube, Google Maps, Google Scholar, and [50+ other engines](https://serpapi.com/search-engine-apis) via the [SerpApi](https://serpapi.com) REST API.

## Quick Start

### 1. Get an API key

Sign up at [serpapi.com](https://serpapi.com/users/sign_up?plan=free) (free tier: 250 searches/month) and set the key:

```bash
export SERPAPI_API_KEY="your_key_here"
```

### 2. Install the plugin

**From the marketplace:**

```
/plugin marketplace add serpapi/serpapi-claude-plugin
/plugin install serpapi@serpapi-plugins
```

**Or test locally:**

```bash
claude --plugin-dir /path/to/serpapi-claude-plugin
```

### 3. Use it

Claude will automatically use SerpApi when you ask it to search for something:

```
Search Google for the best Python web frameworks in 2025
```

```
Compare prices for AirPods Pro on Amazon, Walmart, and eBay
```

```
Find academic papers about transformer architectures published after 2020
```

Or invoke explicitly:

```
/serpapi:search coffee shops near Times Square
```

## What's Included

- **`/serpapi:search`** — A single skill that covers all 50+ SerpApi engines. Claude picks the right engine based on your intent.
- **`engines/`** — Auto-generated JSON schemas for every supported engine, scraped directly from the [SerpApi playground](https://serpapi.com/playground). Claude reads these on-demand for engine-specific parameters.

## How It Works

The plugin teaches Claude to call the SerpApi REST API via `curl`:

```bash
curl -s "https://serpapi.com/search.json?engine=google_light&q=your+query&api_key=$SERPAPI_API_KEY"
```

Every engine follows the same pattern — only the `engine` value and query parameter name change. The skill includes an engine selection table so Claude picks the right one based on your request.

For engine-specific parameters (filters, sorting, pagination), Claude reads the JSON schema from `engines/<engine_name>.json`.

## Keeping Engine Schemas Up to Date

Engine schemas are auto-generated by `build-engines.py`, which scrapes parameter data from the SerpApi playground. A GitHub Actions workflow runs weekly to keep them fresh.

To regenerate manually:

```bash
pip install -r requirements.txt
python build-engines.py
```
